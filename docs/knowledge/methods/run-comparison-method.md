# Run Comparison and Grow History

Run comparison is an evidence review across two to five owned saved grows. It is not a demo-data calculator, a universal score, or proof that one recorded difference caused an outcome.

Start from saved grow records. Preserve the grow, plant, log, task, ToolRun, diagnosis, recipe, telemetry, harvest, dry/cure, pheno and final-quality provenance used for each snapshot. Synthetic QA evidence must remain labeled and must not be mixed into an ordinary production comparison as if it were measured history.

Compare equivalent scopes: whole run, vegetative growth, flowering/fruiting, harvest/final outcome, or post-harvest. When records do not identify a comparable stage or time window, show that gap rather than moving the record into a convenient stage. A user may compare unlike cultivars, phenotypes, crops, plant counts, spaces or methods, but the result must surface those differences as competing explanations.

Do not rank an overall "best run" with hidden weights. The user chooses a reference run and an objective such as yield, final quality, consistency, issue reduction, task execution or cycle time. Report recorded values and deltas only where at least two runs contain comparable evidence. Unknown yield, quality, environment, task, diagnosis or dry/cure data stays unknown rather than becoming zero.

Likely drivers are cautious associations. Each one must include the supporting records, important counter-evidence or alternative explanations, confidence and the next check that could discriminate among them. Use language such as "changed alongside," "may be associated," and "cannot establish causation." One run comparison cannot establish a universal cultivar, environment, nutrition, irrigation or post-harvest method.

The deterministic comparison uses no AI credit. An optional Ask AI step may explain the saved report, but it must receive the same evidence envelope, method/source IDs, missing information and limitations. It may not invent measurements, stage alignment, costs, causes or actions. AI explanation never replaces the saved deterministic report.

Save a `run_comparison` ToolRun and module record containing the selected grow IDs, reference grow, scope, objective, evidence inventory, normalized snapshots, recorded differences, associated-driver hypotheses, missing data, confidence, limitations, recommendations and optional next-run tasks. Owner-reviewed write-backs may create a grow log, tasks and timeline links; comparison itself does not silently change grow targets or SOPs.
